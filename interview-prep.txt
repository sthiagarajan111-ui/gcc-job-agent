NEW ROUTE — GET /api/interview-prep/:jobId
═══════════════════════════════════════════════════════

This route generates interview prep content using
the Claude API for a specific job.

Steps:
1. Find the job by jobId in today's jobs array
2. Check if networking.json already has insights
   for this company — if yes, use them
3. Call Claude API to generate full interview prep
4. Cache the result in memory (prepCache object)
   so same job is not re-generated if clicked again
5. Return the prep data as JSON

Claude API call:
  Model: claude-sonnet-4-20250514
  max_tokens: 1500
  System prompt:
    "You are an expert career coach helping a candidate
    prepare for a job interview. Provide concise,
    actionable interview preparation content.
    Always respond in valid JSON format only.
    No markdown, no code blocks, just raw JSON."

  User message:
    "Generate interview preparation for this job:
    Candidate: Dheeraj Thiagarajan
    Education: LSE MSc Management Finance (Merit) 2025,
    University of Chicago Booth MBA Exchange 2024,
    Newcastle University BSc Business Management 2021
    Experience: Investment & Partnerships Analyst at
    Lovemore Sports (current), Business Development
    at G.Network (65% conversion rate, top 3 by revenue),
    B2B Sales at Durhamlane
    Skills: Financial modelling, B2B sales, C-suite
    stakeholder management, strategy consulting

    Job Title: [title]
    Company: [company]
    Location: [location]
    Industry: [detect from company and title]
    Job Description: [description or 'Not provided']

    Return ONLY this JSON structure:
    {
      companyOverview: {
        whatTheyDo: string (2 sentences),
        size: string (e.g. Large multinational / Mid-size regional),
        culture: string (3-5 culture keywords),
        recentNews: string (1-2 likely recent developments),
        whyDheerajFits: string (2 sentences connecting his background)
      },
      industryInsights: {
        sector: string (industry sector name),
        gccMarketSize: string (market context),
        keyTrends: array of 3 strings (current trends),
        topCompetitors: array of 3 competitor names
      },
      roleInsights: {
        typicalResponsibilities: array of 4 strings,
        kpisMeasuredBy: array of 3 strings,
        careerProgression: string (likely next role),
        salaryRange: string (AED range for this role)
      },
      likelyInterviewQuestions: {
        behavioural: array of 4 question strings,
        technical: array of 3 question strings,
        situational: array of 3 question strings,
        companySpecific: array of 2 question strings
      },
      suggestedAnswerPointers: {
        topStrengthToEmphasise: string,
        starExamples: array of 3 objects each with:
          { situation: string, task: string,

CACHING
═══════════════════════════════════════════════════════
Store generated prep in a module-level object:
  const prepCache = {}

Key: jobId
Value: the full prep JSON object

When route called:
  If prepCache[jobId] exists: return it immediately
  (no API call needed — saves credits)

Also save to file: data/interview-prep-YYYY-MM-DD.json
So prep survives dashboard restart within same day.

On dashboard start, load existing prep from file
into prepCache if file exists for today.
