═══════════════════════════════════════════════════════
THE PROBLEM TO SOLVE
═══════════════════════════════════════════════════════
Job listing URLs go to the job description page, not
directly to the application form. The actual form is
hidden behind a login wall or an Apply button click.

The solution is a 2-phase approach:
Phase 1: Open the job listing page, show a waiting overlay
Phase 2: Detect when user clicks Apply and form appears,
         then auto-fill the form immediately

PHASE 2 — DETECT FORM AND AUTO-FILL
═══════════════════════════════════════════════════════
Add function: async waitForFormAndFill(page, job, portal)

This function polls for the application form to appear
after the user clicks Apply on the listing page.

Logic:
  Set up a polling loop that runs every 1 second
  for up to 60 seconds (60 attempts):

  Each second check:
    1. Has the URL changed? (means a new page loaded)
    2. Are there more than 3 form input fields visible?
       Use: page.$$eval('input:not([type="hidden"])')
    3. Has any of the applyButtonSelectors disappeared?
       (means form replaced the listing)

  When form is detected (any of above conditions true):
    Wait 1.5 seconds for form to fully render
    Call fillAllFields(page) — existing function
    Call injectOverlay(page, filledCount) — existing function
    Update the waiting overlay to show:
      "✅ Form detected and filled! Review yellow fields
      and click Submit when ready."
    Log: "Form detected and auto-filled: X fields"
    Stop polling

  If 60 seconds pass with no form detected:
    Update overlay to show:
      "⏱️ Form not detected automatically.
      If you can see a form, the fields may still
      have been filled. Please check and submit."
    Log: "Form detection timed out after 60 seconds"

UPDATE THE MAIN fillForm FUNCTION
═══════════════════════════════════════════════════════
Replace the current fillForm function with this logic:

async fillForm(url, job):
  1. Detect portal using detectPortal(url)
  2. Log: "Opening [portal.portal] job page..."
  3. Launch Puppeteer with headless: false
     Set defaultViewport: null
     Set args: ['--start-maximized']
  4. Open the URL
  5. Wait for page to load (waitForSelector body, 15s)
  6. Call injectWaitingOverlay(page, portal, job)
     to show Phase 1 instructions
  7. Log: "[portal.portal] detected — waiting for
     user to click Apply button..."
  8. Call waitForFormAndFill(page, job, portal)
     This runs in background watching for the form
  9. Do NOT close browser — keep open for Dheeraj
  10. Return: { portal: portal.portal, browser, page }
