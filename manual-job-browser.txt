THE NEW 3-TIER FALLBACK FLOW
═══════════════════════════════════════════════════════

When user submits a URL, try in this order:

TIER 1 — Direct fetch (no login needed):
  Try axios fetch of the URL
  If successful AND job details extracted: use them
  Done — no browser needed

TIER 2 — Browser with login (if fetch blocked):
  If axios returns 401, 403, or login page detected,
  OR if extracted title is empty/generic:
  Open Puppeteer browser (headless: false, visible)
  Navigate to the URL
  Show overlay: "Please log in to continue.
  The job details will be extracted automatically
  after you log in."
  Wait for user to log in and job page to fully load
  Then extract details from the live browser page
  Close browser after extraction

TIER 3 — Manual entry (if user cancels browser):
  If user closes browser or clicks cancel:
  Show manual entry form in modal
  User types details manually

═══════════════════════════════════════════════════════
LOGIN DETECTION
═══════════════════════════════════════════════════════
Add function: isLoginPage(html, url)
Returns true if the page appears to be a login wall.

Check for these signals (case-insensitive):
  1. URL contains: /login, /signin, /sign-in,
     /auth, /authenticate
  2. HTML contains ALL of these together:
     ("password" AND "email" AND "sign in")
     OR ("password" AND "username" AND "login")
  3. HTML contains: "authwall", "login-required",
     "sign-in-required", "members-only"
  4. Response status is 401 or 403
  5. For LinkedIn specifically:
     HTML contains "authwall" or
     URL contains "linkedin.com/login" or
     URL contains "linkedin.com/authwall"

NEW ROUTE — POST /api/add-job-from-url (UPDATED)
═══════════════════════════════════════════════════════
Replace the existing route handler with this logic:

async (req, res) => {
  const { url, manualData } = req.body

  // If manual data provided, skip everything
  if (manualData && manualData.title) {
    // Go straight to scoring (existing logic)
    return handleManualData(manualData, url, res)
  }

  // TIER 1: Try direct fetch first
  try {
    const response = await axios.get(url, {
      headers: { 'User-Agent': 'Mozilla/5.0...' },
      timeout: 15000
    })

    if (!isLoginPage(response.data, url) &&
        response.status === 200) {
      const extracted = extractJobDetails(response.data, url)
      if (extracted.title && extracted.title.length > 3) {
        // Good extraction — score and return
        return handleExtractedJob(extracted, url, res)
      }
    }

    // If we get here, fetch worked but no good data
    // Fall through to browser approach
  } catch (fetchError) {
    // Fetch failed — fall through to browser approach
  }

  // TIER 2: Need browser — signal client to open browser
  // Return a special response telling client to
  // open browser mode
  return res.json({
    success: false,
    requiresBrowser: true,
    message: "This page requires login. Opening browser...",
    url: url
  })
}

NEW ROUTE — POST /api/add-job-browser
═══════════════════════════════════════════════════════
This route handles the Puppeteer browser extraction.

POST /api/add-job-browser
Body: { url: string }

Steps:

1. Launch Puppeteer with headless: false
   defaultViewport: null
   args: ['--start-maximized']

2. Navigate to the URL
   waitUntil: 'domcontentloaded', timeout: 30000

3. Inject a waiting overlay on the page:
   Position: fixed, top: 0, left: 0, width: 100%
   Background: #1B2A4A (dark navy)
   Color: white
   Padding: 14px 20px
   Z-index: 999999
   Font: Arial 14px
   Text:
     Line 1 (gold, bold): "🤖 GCC Job Agent — Job Extractor"
     Line 2 (white): "If this page requires login,
     please sign in now."
     Line 3 (green): "✅ Job details will be extracted
     automatically once you reach the job page."
     Line 4 (grey, small): "Do not close this browser.
     Return to dashboard after you see the job details."

4. Poll every 2 seconds for up to 120 seconds
   (2 minutes for user to log in):

   Each poll check:
   a. Has the URL changed away from login page?
   b. Does the page now contain job-like content?
      Check for: h1 tag, .job-title, [class*="position"],
      job description keywords in body text
   c. Is the page title different from login page title?

   When job page detected:
     Wait 2 seconds for full render
     Extract page HTML using page.content()
     Also try to extract directly from DOM using
     page.evaluate() for better accuracy:
       title: document.querySelector('h1')?.innerText
       company: multiple selectors tried
       location: multiple selectors tried
       description: longest text block found
     Close the browser
     Return extracted data

5. If 120 seconds pass with no job page:
   Close browser
   Return: { success: false, timedOut: true,
     message: "Browser timed out. Please try
     manual entry." }

6. On success: score and save the job
   Same as TIER 1 success flow

NEW ROUTE — GET /api/browser-status
═══════════════════════════════════════════════════════
Simple status endpoint the client polls:
Store browser extraction status in a module-level
variable: let browserExtractionResult = null

GET /api/browser-status
Returns:
  { status: "waiting" } if still running
  { status: "complete", job: jobObject } if done
  { status: "failed", message: string } if failed
  { status: "idle" } if no browser session active

Reset browserExtractionResult to null after returning
a complete or failed status.
