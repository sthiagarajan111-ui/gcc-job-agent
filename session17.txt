═══════════════════════════════════════════════════════
DATA FILE
═══════════════════════════════════════════════════════
All application data is saved to:
  data/applications.json

Structure of applications.json:
  An array of application objects.
  Create the file with an empty array [] if it
  does not exist.

Each application object has these fields:
  id:              unique string (company_title_date)
  jobTitle:        string
  company:         string
  location:        string
  applyUrl:        string (the job URL)
  tier:            number (1, 2, 3 or 4)
  tierLabel:       string
  totalScore:      number
  salaryAED:       number or null
  appliedDate:     ISO date string (e.g. 2026-03-06)
  status:          string — one of:
                   "Applied", "Screening", "Interview",
                   "Offer", "Rejected"
  lastUpdated:     ISO date string
  followUpSent:    boolean (default false)
  followUpDate:    ISO date string or null
  notes:           string (default empty)
  contactEmail:    string or null (HR contact if found)
  coverLetterPath: string or null (path to .docx file)

═══════════════════════════════════════════════════════
FUNCTIONS TO EXPORT
═══════════════════════════════════════════════════════

1. loadApplications()
   Reads data/applications.json
   Returns array of application objects
   If file does not exist: creates it with [] and returns []

2. saveApplications(applications)
   Writes the full applications array to data/applications.json
   Pretty printed with 2 space indent

3. addApplication(job)
   Input: job object with fields:
     title, company, location, applyUrl, tier, tierLabel,
     totalScore, salaryAED, coverLetterPath
   Creates new application object with all fields above
   Sets status: "Applied"
   Sets appliedDate: today's date
   Sets followUpSent: false
   Checks for duplicates by id before adding
   If duplicate found: log warning and skip
   Saves to applications.json
   Returns the new application object

4. updateStatus(id, newStatus, notes)
   Finds application by id
   Updates status to newStatus
   Updates lastUpdated to today
   Appends notes if provided
   Saves to applications.json
   Valid statuses: Applied, Screening, Interview, Offer, Rejected
   Returns updated application or null if not found

5. getPipelineSummary()
   Returns object with count of applications per status:
   {
     total: number,
     Applied: number,
     Screening: number,
     Interview: number,
     Offer: number,
     Rejected: number,
     tierBreakdown: { tier1: n, tier2: n, tier3: n, tier4: n }
   }

6. async checkFollowUps()
   Loads all applications
   For each application where:
     status is "Applied" (no response yet)
     AND appliedDate was 7 or more days ago
     AND followUpSent is false
   Calls sendFollowUpEmail(application)
   Sets followUpSent: true, followUpDate: today
   Saves updated applications
   Returns: array of applications that got follow-up emails

7. async sendFollowUpEmail(application)
   Sends the follow-up email using nodemailer
   Uses the template above
   To: dheerajt1899@gmail.com
   CC: sthiagarajan111@gmail.com
   Logs: "Follow-up sent for [title] at [company]"
   Returns: true if sent, false if failed

8. getApplicationsByStatus(status)
   Returns array of applications filtered by status

9. getTier1And2Applications()
   Returns all applications where tier is 1 or 2
   Sorted by appliedDate descending (newest first)
