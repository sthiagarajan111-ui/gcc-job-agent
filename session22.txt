7-DAY NETWORKING ACTION PLAN
═══════════════════════════════════════════════════════
Generate a structured day-by-day plan for each job.
This is hardcoded logic — no API call needed.

Day 1 — Research:
  - Research [Company] on LinkedIn and company website
  - Find 3-5 LSE/Newcastle alumni at the company
  - Identify the hiring manager if possible
  - URL: [LSE Alumni Search URL for this company]
  - URL: [Newcastle Alumni Search URL for this company]

Day 2 — Connect with Alumni:
  - Send connection requests to 2-3 LSE alumni found
  - Use Message Type 1 (LSE alumni message)
  - Send connection requests to 2-3 Newcastle alumni
  - Use Message Type 2 (Newcastle alumni message)
  - Do NOT mention the job application yet

Day 3 — Apply:
  - Submit the job application via the apply URL
  - Use Auto-Fill feature from the dashboard
  - Attach CV and cover letter
  - Note the application in the tracker

Day 4 — Follow up on connections:
  - Check if alumni accepted connection requests
  - If accepted: send a follow-up message mentioning
    the application naturally
  - Connect with HR Manager if found via contactFinder

Day 5 — Engage with company content:
  - Like and comment on [Company] recent LinkedIn posts
  - Follow [Company] official LinkedIn page
  - This increases profile visibility to recruiters

Day 6 — Second wave outreach:
  - If no response from alumni: try 1-2 more connections
  - Send cold outreach to HR Manager using Message Type 3
  - Check application status in tracker

Day 7 — Review and escalate:
  - If no response to application: send follow-up email
  - Use the follow-up template from appTracker.js
  - Connect with senior leaders at [Company] on LinkedIn
  - Move to next priority job if no response

DATA FILE
═══════════════════════════════════════════════════════
Save all networking data to: data/networking.json
Structure: array of networking plan objects

Each object:
  jobId:           string
  jobTitle:        string
  company:         string
  location:        string
  tier:            number
  alumniSearchUrls: {
    lse:           string (URL)
    newcastle:     string (URL)
    chicagoBooth:  string (URL)
    lsePage:       string (URL)
    newcastlePage: string (URL)
  }
  linkedInSearchStrings: array of 3 strings
  linkedInMessages: {
    lseAlumni:     string
    newcastleAlumni: string
    coldOutreach:  string
  }
  actionPlan:      array of 7 day objects
  companyInsights: string or null
  generatedDate:   ISO date string

═══════════════════════════════════════════════════════
FUNCTIONS TO EXPORT
═══════════════════════════════════════════════════════

1. generateAlumniSearchUrls(company, location)
   Returns the alumniSearchUrls object with all 5 URLs
   Encodes company name correctly for URLs

2. generateLinkedInSearchStrings(company)
   Returns array of 3 search strings

3. async generateLinkedInMessages(job)
   Calls Claude API 3 times for 3 message types
   Uses claude-sonnet-4-20250514
   max_tokens: 200 each call
   Returns linkedInMessages object
   If API has no credits: returns template placeholders
   with [COMPANY] and [JOBTITLE] to fill manually

4. generate7DayActionPlan(job, alumniUrls)
   Returns array of 7 day objects (hardcoded logic)
   Each day: { day, title, tasks: array of strings,
               urls: array of strings }

5. async generateCompanyInsights(job)
   Calls Claude API once
   max_tokens: 600
   Returns insights string
   If API has no credits: returns null

6. async buildNetworkingPlan(job)
   Master function for one job
   Calls all functions above
   Saves to data/networking.json
   Returns complete networking plan object

7. async buildNetworkingPlansForTier1And2(jobs)
   Filters to tier 1 and tier 2
   Calls buildNetworkingPlan for each
   2 second delay between jobs
   Returns array of networking plans
   Logs summary: "Built networking plans for X jobs"

8. loadNetworkingPlans()
   Reads data/networking.json
   Returns array or [] if not found

9. getNetworkingPlanForJob(company, jobTitle)
   Returns networking plan for specific job or null

═══════════════════════════════════════════════════════
ERROR HANDLING
═══════════════════════════════════════════════════════

  log warning, do NOT crash
- Any function failure: log error, continue to next job
- All async calls in try/catch blocks
