STEP 3 — UPDATE DASHBOARD FOR MULTI-CANDIDATE
═══════════════════════════════════════════════════════

In src/dashboard.js:

1. Load candidates.json on startup:
   const candidates = loadCandidates()
   function loadCandidates() reads data/candidates.json

2. Add candidate filter to filter bar:
   Label: "All Candidates"
   Options:
     All Candidates
     Dheeraj Thiagarajan (blue dot)
     Thiagarajan Shanthakumar (green dot)
   HTML id: "filter-candidate"
   Place as FIRST filter in the filter bar

3. Tag each job with candidateId based on which
   search profile found it OR based on role matching:

   function detectCandidate(job) {
     const title = (job.title || '').toLowerCase()
     const afterSalesKeywords = ['after sales', 'aftersales',
       'service director', 'service manager', 'warranty',
       'general manager', 'head of service', 'director service']
     const isAfterSales = afterSalesKeywords.some(k =>
       title.includes(k))
     if (isAfterSales) return 'thiagarajan'
     return 'dheeraj'
   }

   Call detectCandidate(job) in loadAllJobs() and
   set job.candidateId for every job.

4. Show candidate avatar badge on each job card:
   Small colored circle with initials:
     Dheeraj: blue circle "DT"
     Thiagarajan: green circle "TS"
   Place next to the Tier badge
   Font-size: 11px, width: 28px, height: 28px
   Border-radius: 50%

5. Update applyFilters() to filter by candidateId
   when filter-candidate is not "All Candidates"

6. Update the navbar to show both candidates:
   Replace "Dheeraj Thiagarajan" text in top right
   with: "DT | TS" showing both avatars
