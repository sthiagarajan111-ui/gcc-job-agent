Create ONE file: src/gmailScanner.js

This file scans Dheeraj's Gmail inbox for recruiter reply emails,
auto-detects whether they are interview invites, rejections or offers,
and updates the application tracker status automatically.

Do NOT modify any existing files.

═══════════════════════════════════════════════════════
DEPENDENCIES TO INSTALL FIRST
═══════════════════════════════════════════════════════
Run: npm install googleapis dotenv

═══════════════════════════════════════════════════════
IMPORTANT — HOW GMAIL API ACCESS WORKS
═══════════════════════════════════════════════════════
This file uses the Gmail API with OAuth2.
The credentials file must exist at:
  data/gmail-credentials.json

The token file will be saved at:
  data/gmail-token.json

If gmail-credentials.json does not exist, the file
must log clear setup instructions and exit gracefully
without crashing. Print these exact instructions:

"GMAIL SCANNER SETUP REQUIRED:
1. Go to https://console.cloud.google.com
2. Create a new project called 'GCC Job Agent'
3. Enable the Gmail API
4. Go to Credentials → Create OAuth 2.0 Client ID
5. Application type: Desktop App
6. Download the JSON file
7. Save it as: data/gmail-credentials.json
8. Run node src/gmailScanner.js again to authorize"

On first run after credentials exist:
  Open browser for OAuth consent
  Save token to data/gmail-token.json
  On subsequent runs: use saved token silently

═══════════════════════════════════════════════════════
EMAIL DETECTION KEYWORDS
═══════════════════════════════════════════════════════
Detect email type by scanning subject AND body text.
All matching is case-insensitive.

INTERVIEW keywords (status → "Interview"):
  interview, schedule a call, schedule a meeting,
  would like to meet, invite you to interview,
  next steps, move forward, shortlisted,
  pleased to invite, assessment, video call,
  zoom call, teams call, phone screen,
  we would like to speak, proceed with your application

OFFER keywords (status → "Offer"):
  offer, pleased to offer, job offer, offer letter,
  congratulations, welcome to the team, onboarding,
  start date, salary package, employment contract,
  we are delighted, formal offer

REJECTION keywords (status → "Rejected"):
  unfortunately, regret to inform, not successful,
  not moving forward, other candidates, not shortlisted,
  decided not to proceed, position has been filled,
  not selected, we will not, thank you for applying but,
  keep your cv on file, future opportunities

SCREENING keywords (status → "Screening"):
  screening, initial call, quick call, introductory call,
  HR call, recruiter call, pre-screen, first round,
  preliminary, want to learn more about you

═══════════════════════════════════════════════════════
MATCHING EMAILS TO APPLICATIONS
═══════════════════════════════════════════════════════
To match a Gmail email to an application in
data/applications.json, check these in order:

1. Does the email sender domain match the company name?
   e.g. sender @goldmansachs.com matches "Goldman Sachs"
2. Does the email subject contain the job title?
3. Does the email subject contain the company name?
4. Does the email body contain both the job title
   AND the company name?

If a match is found with confidence: update the status.
If no clear match: log the email as "unmatched" for
manual review — do NOT update any status.

═══════════════════════════════════════════════════════
FUNCTIONS TO EXPORT
═══════════════════════════════════════════════════════

1. async getGmailClient()
   Loads credentials from data/gmail-credentials.json
   If not found: prints setup instructions and returns null
   If token exists at data/gmail-token.json: uses it silently
   If no token: launches browser OAuth flow to get token
   Returns authenticated Gmail API client or null

2. async scanInbox(daysBack)
   Input: daysBack (number, default 7)
   Searches Gmail for emails received in last [daysBack] days
   Uses Gmail search query:
     "in:inbox newer_than:[daysBack]d"
   For each email found:
     Gets subject, sender, body snippet, date
     Calls detectEmailType(subject, body)
     Calls matchToApplication(sender, subject, body)
   Returns array of detected emails:
     { messageId, sender, subject, date, detectedType,
       matchedApplication, bodySnippet }

3. detectEmailType(subject, body)
   Combines subject + body into one string
   Checks against all keyword lists above in this order:
     1. OFFER (check first — highest priority)
     2. INTERVIEW
     3. SCREENING
     4. REJECTION
   Returns one of: "Offer", "Interview", "Screening",
                   "Rejected", or "Unknown"

4. matchToApplication(sender, subject, body)
   Loads applications from data/applications.json
   Tries to match using the 4 strategies above
   Returns matched application object or null

5. async processNewEmails(daysBack)
   Master function — runs the full scan and update cycle
   Calls scanInbox(daysBack)
   For each email where detectedType is not "Unknown"
     AND matchedApplication is not null:
       Calls updateStatus from appTracker.js
       Logs: "✅ Updated [Company] — [JobTitle] to [status]"
   For unmatched emails with a detected type:
       Logs: "⚠️  Unmatched recruiter email: [subject] from [sender]"
   Saves a scan log to: data/gmail-scan-log.json
   Log format: { scanDate, emailsFound, updated, unmatched }
   Returns summary object:
     { scanned, updated, unmatched, offers, interviews, rejections }

6. async getLastScanSummary()
   Reads data/gmail-scan-log.json
   Returns the most recent scan entry or null if no scans yet
